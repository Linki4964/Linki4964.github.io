车联网基础之基础can协议

一、什么是can

是一种专为分布式实时控制设计的串行通信网络。它最早由德国 BOSCH 公司在 20 世纪 80 年代初开发，初衷是为了解决现代汽车中日益复杂的电子控制单元（ECU）之间的点对点连线过多、布线困难的问题.CAN协议的主要作用在于实现车内控制单元(ECU)之间的通讯，而这些ECU被称为CAN节点

CAN 协议在物理层面上使用两根线：CAN High (CAN_H) 和 CAN Low (CAN_L)。
它通过观察这两根线之间的电压差来传输逻辑信号（0 或 1）。这种设计极大地提高了抗电磁干扰的能力，即使在汽车发动机舱这种复杂的电磁环境下也能稳定运行。

二、CAN协议的好处

1.结构简单，部署成本低

CAN 协议最直观的特点就是布线极简。在复杂的系统中（如整车或大型机械），传统点对点通信需要为每个模块拉设独立的线路，这不仅增加了电缆重量，还让维护变得极其困难。

  • 双线制结构：仅需一对绞合线即可连接数十个节点，大幅减少了线束的物理空间占用和材料成本。
  • 节点扩展性强：在总线带宽允许范围内，增加新节点只需并联到总线上，无需重新布线。
  • 减轻重量：使用CAN协议可以降低车辆重量，大约可以减轻20kg

2.协议成熟，易于开发实现

经过几十年的标准化发展，CAN 协议已经形成了一个极其成熟的生态系统：

  • 硬件驱动支持：目前市面上绝大多数的主流 MCU（如 STM32、MSP430、NXP 等）都内置了硬件 CAN 控制器，开发者无需从零编写复杂的逻辑链路层代码。
  • 标准化工具链：从底层的收发器芯片到上层的分析软件（如 CANalyzer、PCAN），开发者可以轻易找到成熟的调试工具和现成的协议栈，极大缩短了产品的研发周期。
  • 集中式诊断：所有的日志可以集成到一条总线上，你只需要一条总线即可收集到每个ECU部件的信息而不再需要一个一个进行收集

3. 极强的抗干扰与鲁棒性

这是 CAN 协议的“看家本领”。工业和汽车环境充满了复杂的电磁噪声，但 CAN 依然能稳如泰山：

  • 差分信号传输：通过 $CAN_H$ 和 $CAN_L$ 的电压差来判断逻辑，能够有效抵消共模干扰。
  • 完善的纠错机制：CAN 协议定义了位填充、CRC 校验、帧格式检查等五种错误检测方法。
  • 故障隔离：一旦某个节点发生硬件损坏并持续报错，CAN 协议的“节点管理”机制会强制该节点退出总线，确保它不会像“害群之马”一样拖垮整个通信网络。

4. 高效的数据交换与实时性

在控制领域，“快”不是唯一的标准，“准时”才是核心：

  • 基于优先级的非破坏性仲裁：通过报文 ID 来决定优先级。即使多个节点同时竞争，高优先级的关键信息（如安全气囊触发信号）也能在微秒级时间内获得响应，且不会造成数据冲突或丢包。
  • 极低的管理开销：CAN 协议的帧头非常精简，在传输小容量控制指令（如传感器数值、电机开关）时，其效率远高于以太网等复杂协议。

CAN协议的物理与链路层

按照 ISO 11898 标准，CAN 协议主要定义了 OSI 七层模型中的最低两层。这种设计使其能够直接控制硬件信号，确保了通信的实时性与极高的鲁棒性。

1.物理层 (Physical Layer)：


can总线的物理层点工艺了电缆的类型，电信号点评，节点要求等标准：

  • 差分信号（Differential Signaling）：CAN 使用两根线（CAN_H 和 CAN_L）的电压差来表示逻辑状态，显性电平 (Dominant)：对应逻辑 0。此时两线压差大，隐性电平 (Recessive)：对应逻辑 1。此时两线压差几乎为 0。
  • 波特率：节点必须通过双线总线连接，波特率最高可达 1 Mbit/s（经典 CAN）或 8 Mbit/s（CAN FD）。
  • 电缆长度：CAN 电缆的最大长度应介于 500 米（125 kbit/s）和 40 米（1 Mbit/s）之间。

2. 据链路层 (Data Link Layer)：

数据链路层负责将原始的位流组织成有意义的报文（Frames），并处理冲突：

  • 封装与拆封：将应用层的数据打包成标准的数据帧。
  • 非破坏性仲裁（CSMA/CA）：当多个节点同时发送数据时，通过逐位比较 ID，优先级高的节点继续发送，优先级低的节点转为接收。
  • 错误检测与报告：通过 CRC 校验、位填充检查等手段确保数据无误。

三、CAN数据帧格式

下面这里会介绍经典CAN的数据帧格式

  1. SOF(Start of Frame)：用于告知其他节点开始通信
  2. ID:用于定义帧的标识符->值越小优先级越高
  3. RTR(Remoete Transimission Request):远程传输控制需求，一个节点是否需要发送数据或者请求数据从其他节点
  4. Control：
  5. Data:包括传输payload，can签名可以从信息中解码
  6. CRC：循环冗余校验，用于校验数据的完整性
  7. ACK：ACK 槽指示节点是否已确认并正确接收到数据。
  8. EOF：EOF 标记 CAN 帧的结束

四、更高层次的CAN协议

  • CAN FD (Flexible Data-rate):随着汽车智能化（如自动驾驶、OTA 远程升级）的发展，传统 CAN 2.0 每帧 8 字节的载荷和 1Mbps 的极限速率已显得捉襟见肘。为此，博世（BOSCH）在 2012 年推出了 CAN FD。
  • CAN XL：这是 CAN 家族的最新成员，数据载荷高达 2048 字节，速率跨入 10Mbps+ 级别，旨在填补 CAN FD 与工业以太网之间的空白。

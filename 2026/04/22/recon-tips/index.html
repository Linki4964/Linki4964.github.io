漏洞赏金新手手册之信息收集篇

前言：许多刚踏入漏洞赏金（Bug Bounty）领域的白帽子常面临一个巨大的困惑：如何高效地进行信息收集？
在学习初期，我们往往沉溺于漏洞原理与利用技巧的钻研，却忽略了“如何发现目标”这一命题。

事实上，在实战挖掘中，信息收集往往占据了我 70% 以上的工作时间——它不仅是发现漏洞的前提，更是决定挖洞上限的关键。
遗憾的是，我发现市面上大多数教程往往只罗列工具，却少有深度探讨“为什么做”以及“怎么做”的底层逻辑。本文将分享我个人总结的信息收集思路，希望能帮助你构建一套属于自己的实战框架。

什么是信息收集（Reconnaissance）？
虽然业界对“信息收集”没有一个死板的标准定义，但我从目的的角度下，我将其定义为：通过各种技术手段，尽可能完整地测绘出目标的资产地图。
这不仅仅是搜集几个域名或 IP 地址，它更像是一场全方位的“资产审计”。你需要挖掘出所有暴露在互联网上的触点，包括但不限于：未公开的子域名与隐藏接口，配置信息，开发者的代码残留（JS 敏感信息、GitHub 泄露）
第三方供应链关联资产等等

为什么信息收集是重中之重？
在漏洞挖掘中，有一个公认的法则：信息差 = 漏洞产出。
避开“内卷”严重的主战场：大多数新手只会盯着主站（如 example.com）猛攻，那里防御最严，也是最多人竞争的。但是通过深度收集找到的一个偏僻子域名，往往防御薄弱，甚至存在低级错误，自然可以函数更多的漏洞。
我自己的经验来讲则是：你发现的资产越多，你的“攻击面”就越大；攻击面越大，你与漏洞之间的距离就越短。

一、了解你的项目

这听起来像是一段废话，但却是无数新手折戟的地方。每一份漏洞赏金项目（Program）都有其独特的“游戏规则”，如果不加辨别地直接开扫，轻则导致提交的漏洞被判定为 N/A（不予接收） 或 Out of Scope（超出范围），重则可能面临账号封禁甚至法律风险。

在正式开始之前，请务必在规则页中寻找以下核心内容：

  • 测试边界：明确哪些域名、IP、子目录或移动端 App 是受保护且允许测试的。不在名单上的资产，哪怕洞再大也不请不要进行攻击。
  • 漏洞收录标准：你需要知道项目方最关心什么？是 SQL 注入、越权访问，还是特定的逻辑漏洞？了解这些能让你事半功倍。
  • 限制以及禁止性行为：该项目是否明确表示了禁止使用某些不允许的攻击方式，是否允许修改其他用户数据？是否允许删除敏感配置？
    奖励机制：了解不同等级漏洞的赏金跨度，有助于你合理分配精力，优先攻克高价值目标。

二、了解你的目标

在写下第一行代码或运行第一个脚本之前，你首先需要搞清楚：你的对手是谁？
我通常会先通过搜索引擎，详尽调研目标的背景。我主要关注于以下信息：

1.业务版图


这家公司是做电商的、云计算的，还是传统制造业？业务属性常常决定了资产分布，通过了解这些你就可以根据目标调整自己的攻击模型了

2.技术基因


通过公司官网的“关于我们”或“人才招聘”，可以了解到公司的业务范围，产品信息等，也可以观察他们招聘的技术栈（如 Java、Python、Go）。这能预示你后续可能遇到的指纹类型，很多白帽子忽略了招聘信息，但那其实是合法且免费的内部架构说明书。

3.体验功能

你完全可以进入这个网站，尝试注册一个账号，使用他们给你提供的服务，体验他们的产品，这种“沉浸式”体验能让你对目标的逻辑结构有更深刻的肌肉记忆

三、根域名收集

从这一部分开始我们就可以进行我们所熟悉的信息收集部分了，在确定了目标的背景后，第一步任务就是：枚举出目标旗下的所有根域名（Seed Domains）与 IP 资产。这一步决定了你后续扫描的“地基”是否稳固。我收集根域名的途径主要有以下这些：

1.官方与企业关联查询

  • 项目公告：这是最稳妥的来源。项目方通常会列出已知的根域名。注意： 如果公告里写的是 *.example.com，记得去查一下 example.net 或 example.cn 是否也在范围内，有时会有意外收获。
  • 企业信息平台：对于大型企业，其业务往往分布在不同的子公司下。通过 企查查、天眼查、小蓝本 等平台，查询目标的“股权穿透图”或“网站备案信息”，可以挖掘出大量容易被忽视的边缘业务域名。
    小蓝本：https://www.xiaolanben.com/

2.ASN 资产测绘

什么是 ASN？
ASN (Autonomous System Number，自治系统号) 简单来说就是互联网中的“行政区划”。大型企业（如阿里巴巴、腾讯、字节跳动）通常会申请属于自己的 ASN，用来管理其庞大的 IP 地址池。

通过查询 ASN，你可以绕过复杂的域名关联，直接定位到目标企业租用的整个 IP 段。这意味着你能发现那些还没来得及解析域名、仅通过 IP 访问的测试环境。
常用查询途径：

  • 网站查询：
    事实上我常常使用这些网站去查询,他们其实比工具搜集要准确，而且也少很多脏数据

    bgp tool：https://bgp.tools/

    bgp.he：https://bgp.he.net/
  • 工具查询：
    当然如果你想批量获取这些ASN的信息，OWASP Amass 是目前的行业标杆。它不仅能查 ASN，还能自动建立资产关联地图。
    amass:https://github.com/owasp-amass/amass.git

3.WHOIS反查

什么是whois
WHOIS 是一种用于查询 互联网资源注册信息 的协议或服务，通过它可以查到某个 域名、IP地址或 ASN（自治系统编号） 的注册者、管理机构以及相关联系信息。简单来说，whois是一份域名的“房产证”。它记录了域名的注册人姓名、注册邮箱、联系电话以及注册时间。

企业在注册域名时，往往会使用统一的企业邮箱或法人信息。通过“反查”这些关键字段，你可以发现或许你可以在这里发现一些他们从未使用过的域名，并且这些域名也运行着一些被遗忘的服务

网页查询：

whoxy：https://www.whoxy.com/
爱站网：https://whois.aizhan.com/reverse-whois?frm=deadnine.comhttps://whois.aizhan.com/reverse-whois?frm=deadnine.com

4.搜索引擎

空间测绘工具：
空间测绘工具就像是互联网的搜索引擎，但它们抓取的不是网页内容，而是服务指纹、证书信息和端口响应。
但是要注意一点就是空间测绘的信息具有一定的滞后性
fofa：https://fofa.info/

shodan：https://www.shodan.io/
hunter：https://hunter.qianxin.com/
谷歌语法
我常常利用谷歌语法搜索相关的网页的特征信息来获取更多的网站比如我会搜索隐私条例（Privacy policy text）,ICP备案,版权信息等信息，这些信息来获得更多的网页资产

四、子域名枚举

当我们获得了足够的根域名以后我们可以进行子域名枚举，子域名枚举我常常分为三个步骤分别是子域名抓取,页面链接抓取抓取,子域名爆破

1.子域名抓取

利用第三方数据库（如DNS 历史记录）获取信息，优点是速度极快且不触发目标警报。
subfinder：https://github.com/projectdiscovery/subfinder.git
由projectdiscovery团队开发的子域名收集工具他可以抓取互联网上的子域名历史记录，同时你也可以配置其它API信息他就可以进行其它网络信息进行配置


通过修改.config/subfinder/provider-config.yaml添加各个平台的API密钥
通过subfinder -vAPI配置情况
amass：是一个十分强大的子域名搜集工具，同时他也集成了子域名爆破拓展

2.网页链接抓取

很多“边缘资产”并没有解析记录，但它们隐藏在主站的 JS 文件、注释或侧边栏链接中。当你搜集到一些子域名的时候常常使用爬虫工具获取这些网页上的链接，可以获得很多的子域名信息

gospider：这是一款十分好用的爬虫工具，而且会帮你贴心的将信息进行分类，可以减少脏数据的影响

burp suit：这bp上也有十分强大的爬虫工具，使用scan的sitemaps功能可以获得网页url的信息

3.子域名爆破

往往有很多时候，有一些子域名并不会被暴露在DNS服务器上，因此我们可以主动向目标发起询问以便来获得更多的子域名信息

amass：就像上面说的amass已经集成了爆破功能了，因此你可以使用该功能进行爆破

shuffleDNS ：这款工具也是有project discovery开发的一款工具，极速爆破专家，支持自定义解析器以防被运营商 DNS 缓存污染

字典
域名爆破的时候有一个合适的字典其实相当的重要，因为这往往影响你发现的准确性和效率，通常情况下我一般把字典分为两种通用字典，自定义字典

  • 通用字典：顾名思义就是一个集合了各种域名可能性的超级大字典，你可以使用这些字典进行全面的探索和搜查，但是这种字典有时候也往往会花费更长的时间
    aseetnote：https://wordlists.assetnote.io/
    seclist：https://github.com/danielmiessler/SecLists
  • 自定义字典：当然你也可以根据你的目标生成你想要的字典，但是这样生成的字典可能需要一些技巧和经验
    alterx：https://github.com/projectdiscovery/alterx.git
    同样是由pdtm开发的工具，他会通过模式学习生成动态字典。它能根据已发现的域名，自动推测可能的变体

五、其它什么的

当你收集到了足够的子域名以后，我们就可以获取更多关于目标的相关信息，我通常都会思考以下的一些内容

1.端口扫描

除了常见的 80/443，许多敏感服务（如 Redis、Jenkins、数据库）往往运行在非标端口。
naabu：ProjectDiscovery 开发的极速端口扫描器，支持与 subfinder 无缝联动。

nmap：端口扫描界的“瑞士军刀”，适合对重点目标进行深度服务识别和脚本探测

2.网页截图

面对几百个 Web 页面，一张张截图能让你在几分钟内通过视觉快速定位到：后台管理、未授权面板、报错页面、或是有趣的开发环境。

HTTPscreenshot：https://github.com/breenmachine/httpscreenshot.git

3.指纹扫描与技术栈识别

识别出目标使用了什么框架（如 Spring Boot）、什么中间件（如 Shiro）或什么 CMS（如 WordPress），是进行针对性攻击的前提。
httpx：这是我最常用的工具，它可以一次性获取状态码、标题、技术栈和指纹。

Awesome-hacking-tools：https://github.com/awake1t/Awesome-hacking-tools.git
这是一个继承了大量国产设备指纹信息的扫描工具，更适合国内的漏洞挖掘，强烈推荐

4.弱口令尝试

BruteSpray：能接收 Nmap 或 Naabu 的输出，自动识别出开放的协议（如 SSH, FTP, MySQL, MSSQL），并调用 Medusa 进行默认/弱口令爆破。

3.笔记记录
面对海量的域名、IP 和扫描结果，如果只是存在文本文件里，很快就会陷入混乱。高效的白帽子，一定是优秀的“情报管理专家”。
我建议你可以使用思维导图或者文字笔记进行记录你的工作进度，这有助于对你都工作记录进行实时的跟踪，它可以帮你梳理目标的业务版图，确保你没有遗漏任何一个可能的切入点。

查看用户的token是JWT格式

JWT（JSON Web Token）是一种基于JSON的开放标准（RFC 7519），用于在各方之间安全地传输紧凑、自包含的身份验证和授权信息。JWT 将用户信息加密封装在Token中，通过签名保证安全性，服务器无需在服务端存储会话（Session），适用于前后端分离和微服务场景。

JWT 的核心结构与特性
JWT 通常由三部分组成，用圆点（.）分隔：

  • Header（头部）：声明签名算法和Token类型。
  • Payload（负载/有效载荷）：包含声明（Claims），即实际数据，如用户ID、角色、过期时间等。
  • Signature（签名）：对前两部分进行加密，用于验证数据的真实性和完整性。

我们使用jwt解码器查看可以获得这个JWT的信息，我们需要把role换成admin就可以欺骗系统了，但是JWT是需要有签名进行验证的因此我们除非知道密钥的内容是什么否则我们无法伪造密钥

方法一：None JWT攻击

但是我们可以通过将alg修改成来将JWT加密算法制空来进行绕过加密验证

JWT None攻击：JWT 支持将算法设定为 “None”。如果“alg” 字段设为“ None”，那么签名会被置空，这样任何 token 都是有效的。 设定该功能的最初目的是为了方便调试。但是，若不在生产环境中关闭该功能，攻击者可以通过将 alg 字段设置为 “None” 来伪造他们想要的任何 token，接着便可以使用伪造的 token 冒充任意用户登陆网站。

参考：https://portswigger.net/kb/issues/00200901_jwt-none-algorithm-supported
